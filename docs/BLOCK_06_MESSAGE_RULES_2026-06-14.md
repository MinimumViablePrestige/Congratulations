# Block 6: жесткие правила блока поздравлений

Дата: `2026-06-14`

## Что сделали

1. Вынесли правила раскладки поздравлений в отдельный слой.
2. Для каждого режима задали собственные ограничения:
   - сколько карточек помещается на страницу
   - сколько символов безопасно помещается в одну карточку
3. Подключили эти лимиты к валидации:
   - при добавлении поздравления участником
   - при редактировании поздравления организатором
4. Убрали внутреннюю прокрутку, из-за которой на финальном экране появлялись половинчатые карточки.
5. Перевели блок поздравлений на пролистывание целыми страницами.
6. Для `Сетки 2 на 2` заложили расширяемую логику:
   - в обычном случае страница содержит `2x2`
   - если вокруг мало других блоков, сетка может расширяться до `2x3`

## Почему это важно

1. Финальная открытка больше не должна показывать половину следующего поздравления.
2. Карточки в каждом режиме становятся визуально честными и предсказуемыми.
3. Лимиты теперь известны не только коду, но и пользователю через интерфейс.

## Что еще требует живого теста

1. Насколько комфортен новый вариант `2 ряда` после выравнивания карточек.
2. Достаточно ли текущих лимитов по символам для реальных поздравлений.
3. Когда именно расширенную сетку `2x3` стоит включать автоматически.
